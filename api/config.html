Application Config API¶

  • Config Methods
  • Config Features

Use application config API to modify and access the application configuration.

Config Methods¶

Note

The configuration is read-only after an application is fully bootstrapped.

  • $app->getConfig()
  • $app->getCoreConfig()

$app->getConfig()¶

It is possible to set configuration values before an application bootstrap.

Setting a configuration value.

$app->getConfig()->myConfigKey = 'myConfigValue';

Getting a configuration value.

/** @var \Zend\Config\Config|mixed $myConfig */
$myConfig = $app->getConfig('myConfigKey', $default = null);

$app->getCoreConfig()¶

Getting a core configuration value.

/** @var \Zend\Config\Config|mixed $myCoreConfig */
$myCoreConfig = $app->getCoreConfig('myCoreConfigKey', $default = null);

Config Features¶

Config features are registered into the config like this:

use WebinoAppLib\Feature\Config;

new Config([

    new SomeConfigFeature, // <--

]);

Available features:

  • Events
    • Listener
    • CoreListener
  • Services
    • Service
    • CoreService
  • Logging
    • Log
    • FirePhpLog
  • Cache
    • ConfigCacheEnabled
    • FilesystemCache
    • MemoryCache
  • Filesystem
    • DefaultFilesystem
    • MemoryFilesystem

Events¶

Use event system config features to configure listeners.

Listener¶

  • namespace: WebinoAppLib\Feature

Application listener config feature.

use WebinoAppLib\Feature\Config;
use WebinoAppLib\Feature\Listener;

new Config([

    new Listener(MyExampleListener::class), // <--

]);

CoreListener¶

  • namespace: WebinoAppLib\Feature

Application core listener config feature.

use WebinoAppLib\Application\CoreConfig;
use WebinoAppLib\Feature\CoreListener;

new CoreConfig([

    new CoreListener(MyExampleCoreListener::class), // <--

]);

Services¶

Use services config features to configure invokable services and service factories.

Service¶

  • namespace: WebinoAppLib\Feature

Application service config feature.

use WebinoAppLib\Feature\Config;
use WebinoAppLib\Feature\Service;

new Config([

    new Service(MyExampleService::class), // <--

]);

Obtaining service instance.

$service = $app->get(MyExampleService::class); // <--

CoreService¶

  • namespace: WebinoAppLib\Feature

Application core service config feature.

use WebinoAppLib\Application\CoreConfig;
use WebinoAppLib\Feature\CoreService;

new CoreConfig([

    new CoreService(MyExampleCoreService::class), // <--

]);

Obtaining core service instance.

$service = $app->get(MyExampleCoreService::class); // <--

Logging¶

Use logging config features to configure logging.

Log¶

  • namespace: WebinoAppLib\Feature

Logging config feature.

use WebinoAppLib\Application\CoreConfig;
use WebinoAppLib\Feature\Log;

new CoreConfig([

    new Log, // <--

]);

Logging a message class.

$app->log(MyDebugMessage::class);

FirePhpLog¶

  • namespace: WebinoAppLib\Feature

FirePHP logging config feature.

use WebinoAppLib\Application\CoreConfig;
use WebinoAppLib\Feature as AppFeature;

new CoreConfig([

    new AppFeature\Log,
    new AppFeature\FirePhpLog, // <--

]);

Cache¶

Use cache config features to configure caching.

ConfigCacheEnabled¶

  • namespace: WebinoConfigLib\Feature

Application config cache enabled config feature, it toggles application config caching.

use WebinoAppLib\Feature\Config;
use WebinoConfigLib\Feature\ConfigCacheEnabled;

new Config([

    // enable
    new ConfigCacheEnabled, // <--

    // disable
    new ConfigCacheEnabled(false), // <--

]);

Obtaining cache storage instance. [TODO]

$app->cache();
// or
$app->cache(DefaultCache:::class);

FilesystemCache¶

Application filesystem cache config feature.

use WebinoAppLib\Application\CoreConfig;
use WebinoAppLib\Feature\FilesystemCache;

new CoreConfig([

    new FilesystemCache, // <--

]);

MemoryCache¶

  • namespace: WebinoAppLib\Feature

Application memory cache config feature.

use WebinoAppLib\Application\CoreConfig;
use WebinoAppLib\Feature\MemoryCache;

Filesystem¶

Use filesystem config features to configure filesystems.

DefaultFilesystem¶

  • namespace: WebinoAppLib\Feature
  • scope: Local working dir.

Overriding an application default filesystem.

use WebinoAppLib\Application\CoreConfig;
use WebinoAppLib\Feature\DefaultFilesystem;

new CoreConfig([

    // optional
    new DefaultFilesystem('custom/root'), // <--

]);

Obtaining default filesystem instance.

use WebinoAppLib\Filesystem\LocalFiles;

$defaultFilesystem = $app->files();
// or
$defaultFilesystem = $app->files(LocalFiles::class);

MemoryFilesystem¶

  • namespace: WebinoAppLib\Feature
  • scope: Temporary filesystem in the memory

Application filesystem in the memory.

use WebinoAppLib\Application\CoreConfig;
use WebinoAppLib\Feature\MemoryFilesystem;

new CoreConfig([

    new MemoryFilesystem, // <--

]);

Obtaining memory filesystem instance.

use WebinoAppLib\Filesystem\InMemoryFiles;

$memoryFilesystem = $app->files(InMemoryFiles::class);
